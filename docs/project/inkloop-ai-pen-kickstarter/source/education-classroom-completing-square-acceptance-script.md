# 教育课堂“配方法”人工验收脚本

## 这份脚本验收什么

固定教学题目：用配方法解方程 `x² + 4x - 5 = 0`。

本脚本故意把结果分成两层，不能混为一个“通过”：

1. **流程通过**：两个 Web 端能同步板书，学生能触发当前步骤/框选解释、课后总结和练习，结果能定位来源，教师能审核 `LessonGraph`。
2. **数学语义通过**：系统真的认出了公式，能正确解释配方法、生成正确总结和同类练习。

当前实现会把课本页、教师确认/更正的公式、final/corrected 教师字幕和板书来源一起送入 AI。只有 `execution_mode=real` 且通过本文数学检查的结果才算语义通过；fixture 或确定性降级只证明工作流与容错。

## 验收前准备（2 分钟）

- 教师 Web：`https://<Mac局域网IP>:8872/teacher-classroom.html`
- 学生 Web：`https://<Mac局域网IP>:8872/student-classroom.html`
- 使用两个浏览器窗口；最好一个普通窗口做教师端，一个无痕窗口做学生端。
- 教师端和学生端都应只有一个无限点阵教学视口；教材、笔迹和焦点一起平移/缩放，教材层内部不能滚动。
- 教师端画布应能正常画线；学生端应显示“实时同步”。
- 本轮不使用电子纸设备。
- 记录 AI 结果右上角是“真实 AI”还是“确定性降级”。确定性降级只能证明容错，不能证明数学语义。

画布布局建议：题目写在教材右侧的页外点阵区，五个步骤一行一个，行间留出至少一个字符高度。这样既能验收页外书写，也方便学生准确框选第 ③ 行。内置讲义和自动化 fixture 使用同一个方程。

开始前再验收一次视角：教师缩放或平移时，跟随学生应同步；学生翻页后进入“自由浏览”，教师继续移动时学生只看到提示而不被拉回；点击“返回老师位置”后精确恢复教师页码、中心和缩放。

## 逐分钟教学与操作

### 0:00–1:00 创建并加入课堂

教师端：

1. 课堂名称填“配方法验收课”。
2. 点击“创建课堂”。
3. 把页面上的六位课堂码给学生端。
4. 点击“开始上课”。顶部应从“等待开始”变为“正在授课”。

学生端：

1. 输入课堂码。
2. 昵称填“学生A”。
3. 点击“进入课堂”。
4. 确认顶部显示“实时同步”，右侧有“框选板书范围”和“解释当前步骤”。

通过条件：学生端进入了同一课堂，AI 区域仍是学生自己的私人空间。

### 1:00–3:00 写题目与前两步

教师一边写，一边按下面原话讲。若使用可信 HTTPS 课堂入口，这段讲话用于实时音频和字幕；HTTP 开发入口只验收课本、板书和 AI 工作流。

| 顺序 | 教师写在画布上的内容 | 教师说的话 | 写完后的动作 |
| --- | --- | --- | --- |
| 题目 | `x² + 4x - 5 = 0` | “今天我们用配方法解这个一元二次方程。” | 停 2 秒 |
| ① | `x² + 4x = 5` | “第一步，把负五移到等号右边，右边变成五。” | 停 2 秒 |
| ② | `x² + 4x + 4 = 9` | “四的一半是二，二的平方是四，所以等式两边同时加四。” | 停笔，不继续写 |

每写完一行，观察学生端是否出现同样的笔迹。不要追求字迹漂亮，但 `x²`、正负号和等号要分开写清楚。

### 3:00–4:00 验收“解释当前步骤”

学生端不要框选，直接点击“解释当前步骤”。等待结果出现。

流程通过：

- 出现“正在解释当前板书…”或排队/运行状态，随后产生一个结果。
- 结果标明“真实 AI”或“确定性降级”。
- 每个结果段至少有一个“来源”按钮；点击后教师最近的笔画会被定位或强调。
- 结果只出现在学生端，不改变教师白板和教师规范课堂笔记。

数学语义通过：结果至少明确表达以下三点，且不能把错误步骤说成正确步骤：

- 当前是在配完全平方。
- 因为一次项系数是 `4`，所以添加 `(4/2)² = 4`。
- 等式两边都加 `4`，所以右侧从 `5` 变为 `9`。

若结果只描述笔画、线条、顺序或声称公式无法确认：流程可通过，数学语义不通过。

### 4:00–5:00 写第三步并验收“框选区域”

教师写：

```text
③ (x + 2)² = 9
```

教师说：“左边现在可以写成 x 加二的平方，这就是完全平方式。”停笔。

学生端：

1. 点击“框选板书范围”。
2. 在画布上拖动矩形，只包住第 ③ 行，不要包含第 ② 或第 ④ 行。
3. 页面提示“已框选板书范围，可以触发‘解释当前步骤’”后，点击“解释当前步骤”。

流程通过：

- 框选框可见，且请求完成后消失或重置。
- 结果来源只定位到框选相交的板书笔画，不应定位到画布顶部无关区域。
- 若框得过小，页面提示“框选范围太小”，可重新框选。

数学语义通过：结果应说明 `x² + 4x + 4` 等于 `(x + 2)²`，而不是解释移项或直接给出最终答案。

### 5:00–7:00 完成解题

教师继续逐行书写并讲解：

| 顺序 | 教师写在画布上的内容 | 教师说的话 | 写完后的动作 |
| --- | --- | --- | --- |
| ④ | `x + 2 = ±3` | “两边开平方时要保留正负两个分支，所以 x 加二等于正负三。” | 停 2 秒 |
| ⑤ | `x = 1 或 x = -5` | “分别解两个一次方程，得到 x 等于一或负五。” | 停 2 秒 |

可选人工核算：把 `1` 和 `-5` 分别代回原方程，两者结果都为 `0`。

### 7:00–8:00 结束课堂

教师点击“结束课堂”，在确认框中确认。

通过条件：

- 教师端显示“课程已结束”，画布不能再新增笔画。
- 学生端自动显示“课程已结束”。
- “解释当前步骤”和框选关闭，“生成完整课堂总结”和“生成课后练习”开放。

### 8:00–10:00 验收“课后完整总结”

学生点击“生成完整课堂总结”。

流程通过：

- 任务显示状态并最终产生结果。
- 总结包含多个有来源的段落，来源可回到板书。
- 刷新学生页面后，该学生仍能看到自己的总结。

数学语义通过：总结必须按顺序覆盖：

1. 原方程 `x² + 4x - 5 = 0`。
2. 移项得到 `x² + 4x = 5`。
3. 两边加 `4` 得到 `x² + 4x + 4 = 9`。
4. 配成 `(x + 2)² = 9`。
5. 开平方保留 `±`。
6. 最终得到 `x = 1` 或 `x = -5`。

只要遗漏“等式两边同时加 4”、漏掉负分支或给出错误答案，数学语义即不通过。

### 10:00–12:00 验收“课后练习”

学生点击“生成课后练习”。

流程通过：

- 结果明确分为“题目 / 提示 / 答案”。
- 三部分都有来源，且只在该学生端可见。

数学语义通过：至少给出一道同类的可解题目。推荐期望题目：

```text
x² + 8x + 7 = 0
```

推荐提示：移项后两边加 `(8/2)² = 16`，配成 `(x + 4)² = 9`。

正确答案：

```text
x = -1 或 x = -7
```

题目不必与推荐题完全相同，但必须能用本课的配方法解决，提示不能直接泄露全部答案，最终答案必须正确。

### 12:00–15:00 验收教师 LessonGraph 审核

教师端在“教师规范课堂笔记”区域点击“生成课堂候选”。

先观察：

- 候选是否按步骤编号显示。
- 每项是否有内容、状态和“定位来源”。
- 低置信公式是否显示“低置信公式 · 需要审核”。
- 页面标明“真实 AI 候选”或“确定性降级候选”。

然后故意完成三种审核动作：

1. 对一项正确候选点击“接受”。
2. 对一项不完整候选点击“编辑”，改成“等式两边同时加 4，得到 x² + 4x + 4 = 9”。
3. 对一项与解题无关或错误的候选点击“驳回”。如果没有错误项，可选择最弱、重复的一项来验证驳回流程，并在记录中注明这是流程测试。
4. 其余候选逐项接受、编辑或驳回，直到显示“审核已完成”。

流程通过：

- 三种动作都能保存。
- 所有候选处理后出现“已审核课堂步骤”。
- 刷新教师页面后，审核状态和最终步骤仍在。
- 点击来源能定位对应板书。
- 学生之前的解释、总结和练习不会自动进入教师 `LessonGraph`。

数学语义通过：最终“已审核课堂步骤”按正确逻辑保留配方法主线，不包含错误公式，并以 `x = 1 或 x = -5` 收束。教师手工编辑后正确不等于自动识别通过；请在记录中分别注明“AI 原始候选质量”和“教师审核后质量”。

## 最终判定

- **第一阶段功能验收通过**：上表六项的“流程通过”全部勾选，来源跳转、刷新恢复和学生私有边界正确。
- **第一阶段数学理解不通过但符合现状**：流程全部通过，但一个或多个“数学语义通过”未勾选；把原始错误保留下来，作为下一阶段固定回归样本。
- **下一阶段硬门槛通过**：在加入课本、手写/公式识别和教师字幕后，除流程全部通过外，五个 AI 原始结果也应达到上述数学语义标准；教师不应靠重写全部内容才能得到正确 `LessonGraph`。
- **直接判定故障**：教师无法写、学生没有同步、按钮状态错误、任务永远不结束、来源跳转到无关区域、刷新后结果丢失、学生结果出现在教师或其他学生端。
