# 硬件采购清单

这份文档是飞书多维表格的建表规范。正式采购数据统一放在飞书多维表格里，本地保留字段设计、视图规则、样例行和采购判断口径。

## 建表目标

硬件采购单要同时服务四件事：

1. 6 月底完成硬件采购清单，明确要买什么、为什么买、谁负责、什么时候下单。
2. 7 月底支撑硬件选型和设计冻结，能看到供应商、交期、风险和备选方案。
3. 8 月底支撑样机功能验证和装配，能看到物料到货、兼容性、问题和决策。
4. 9 月底支撑最终样机和最终软件实测，能回看哪些硬件路线可量产，哪些要淘汰。

## 多维表结构

建立 4 张表：

| 表名 | 用途 | 是否必须 |
| --- | --- | --- |
| 采购主表 | 所有物料、设备、模组、服务、样品的采购和验证状态 | 必须 |
| 供应商表 | 供应商、联系人、报价、能力和风险 | 必须 |
| 询价记录 | 每次询价、报价、样品沟通、商务条件记录 | 选填 |
| 验证记录 | 到货后硬件、电气、结构、软件适配、样机验证记录 | 选填 |

### 采购目的

| 选项 | 说明 |
| --- | --- |
| MVP 验证 | 7 月中旬第一版 MVP 所需 |
| EVT | 工程验证测试 |
| 样机装配 | 8 月底样机功能验证和装配 |
| 量产 RFQ | 用于量产报价和供应商比较 |
| 备选方案 | 主路线失败时替代 |
| 工具耗材 | 调试、装配、测试工具 |

### 决策状态

| 选项 | 含义 |
| --- | --- |
| 待锁定 | 需求或规格未定 |
| 待询价 | 规格已定，待联系供应商 |
| 待下单 | 报价已拿到，等待决策 |
| 已下单 | 已采购但未到货 |
| 已到货 | 已收到实物 |
| 验证中 | 正在做硬件/软件/结构验证 |
| 通过 | 可进入下一阶段 |
| 淘汰 | 不再使用 |
| 暂缓 | 暂不采购或暂不验证 |

### 关键风险

| 选项 | 说明 |
| --- | --- |
| 交期风险 | 影响 MVP 或样机时间 |
| 成本风险 | 单价或 MOQ 过高 |
| 兼容风险 | 接口、电气、结构或软件不确定 |
| 驱动适配风险 | BSP、TCON、waveform、触控驱动不明确 |
| 供应商风险 | 供应商响应慢、资质不清、报价不稳定 |
| 量产风险 | 样品可买但量产不可控 |
| 认证风险 | 出海认证资料不足 |
| 体验风险 | 延迟、残影、手写体验或功耗不达标 |

## 维护规则

- 每周周会前更新 `决策状态`、`风险等级`、`期望到货日期` 和 `验证结论`。
- P0 物料必须有备选方案，且不能只写“找替代”。
- 高风险物料必须关联一个下一步动作和负责人。
- 样机阶段所有到货物料必须补一条验证记录。
- 进入 7 月底硬件设计冻结前，所有 P0/P1 物料必须明确：采用、备选、淘汰或暂缓。
