## 问题与修正

| 问题 | 原因 | 修正 |
| --- | --- | --- |
| 飞书显示 Markdown 原文 | 直接写入 docx 文本块不会渲染 Markdown | 改用飞书官方 Markdown 导入 |
| 页面标题重复 | Markdown H1 与飞书页面标题同时显示 | 飞书导入副本去掉正文开头标题 |
| 部分文档仍是模板 | 原飞书导出带有模板占位内容 | 改写为项目正式内容 |
| 技术文档过多 | 下载目录资料来源分散 | 建立技术方案总览和资料归档索引 |

## 经验总结

1. 飞书同步不适合把 Markdown 当普通文本写入；需要通过官方导入链路生成原生块。
2. 本地 Markdown 和飞书展示需要不同处理：本地保留 H1，飞书正文去掉开头 H1。
3. 项目管理文档不能保留模板占位；即使是第一版，也必须写成可直接执行的内容。
4. 技术方案长文保留原貌，但必须有总览文档帮助决策和检索。
