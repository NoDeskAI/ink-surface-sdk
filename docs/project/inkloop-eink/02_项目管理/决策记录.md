## ADR-001 文档库采用本地 Markdown 优先

- 日期：2026-06-29
- 状态：已采用
- 背景：飞书适合协作，但项目资料散落在飞书和下载目录。本地 Markdown 更适合持续编辑、版本管理和批量整理。
- 决策：先建立桌面本地文档库，所有新项目资料优先写 Markdown，再同步飞书。
- 影响：需要维护本地到飞书的同步规则，避免双向编辑造成版本冲突。
