# 彩色电子纸硬件出海产品立项调研 v0.9

调研日期：2026-06-16

## 1. 调研口径

本报告主攻硬件调研，目标是梳理一款面向海外市场的高端彩色电子纸标注设备需要哪些核心硬件单元、各单元如何配合、国内供应链如何完成原型打样。

当前产品假设：

- 产品方向：高端纸质感、彩色电子纸、手写标注、PDF/文档阅读。
- 屏幕尺寸：优先看 10.3-11.8 英寸，兼顾 13.3 英寸大屏可能性。
- 供应链：优先调研国内可打样、可小批量、可量产协同的路径。
- 淘宝/1688：用于验证原型组件可得性、模块组成和早期 BOM 区间，不直接等同于量产供应链结论。

云端软件和 AI 功能在本文中只作为硬件需求来源处理：

- 如果需要云同步，需要什么通信和存储硬件。
- 如果需要云端 AI，需要什么联网、缓存、输入和安全硬件。
- 如果需要部分本地 AI，需要什么 NPU、内存、散热和电源支持。
- 本文不展开订阅、商业化、增长、内容生态等软件业务问题。

## 2. 整机硬件单元

| 序号 | 单元 | 主要组成 | 关键作用 | 重点验证 |
|---|---|---|---|---|
| 1 | 显示单元 | 彩色电子纸面板、EPD 驱动/TCON、波形文件、前光、导光板、盖板玻璃/膜 | 决定纸质感、色彩、刷新、残影、可读性 | 面板尺寸、白底、彩色标注、局刷、前光均匀性 |
| 2 | 触控/笔单元 | 电磁笔或主动笔、电容触控层、压感、橡皮擦、侧键、笔芯、掌托误触 | 决定标注交互体验 | 采样率、延迟、视差、摩擦感、笔迹稳定性 |
| 3 | 主控单元 | SoC、RAM、eMMC/UFS、PMIC、USB-C、时钟、按键、霍尔、IMU | 决定系统性能、待机功耗、驱动适配 | 电子纸驱动、PDF 性能、休眠唤醒、BSP 成熟度 |
| 4 | 通信单元 | Wi-Fi、Bluetooth、可选蜂窝/eSIM、天线 | 支持同步、OTA、云端 AI、蓝牙配件 | 弱网同步、待机联网功耗、天线布局、认证 |
| 5 | AI/算力单元 | 无独立 AI、SoC 内置 NPU、外挂 AI 加速器 | 支持本地 OCR、离线识别、端侧预处理 | NPU 是否必要、模型效果、推理功耗、散热 |
| 6 | 电源单元 | 软包锂电池、保护板、充电 IC、fuel gauge、电源路径管理 | 决定厚度、重量、续航、充电体验 | Wh、休眠电流、充电温升、续航模型 |
| 7 | 安全/存储单元 | eMMC/UFS、安全芯片或 TEE、加密存储 | 支持本地文件、云缓存、账号凭据、隐私文件 | 存储寿命、加密性能、数据擦除 |

## 3. 云端功能对应的硬件需求

云端 AI 本身不是本文重点，但它会反向决定设备要不要增加通信、输入、缓存、安全和本地算力单元。

| 云端/AI 功能 | 必需硬件 | 可选增强硬件 | 说明 |
|---|---|---|---|
| 账号登录、文件同步、OTA | Wi-Fi、eMMC、RTC、电源管理 | Bluetooth 配网、手机 App 辅助配网 | Wi-Fi 是基本要求；蓝牙主要用于配网和配件连接 |
| PDF/笔记上传云端 AI 总结 | Wi-Fi、足够本地存储、稳定电源管理 | 安全芯片/TEE、硬件加密 | 需要本地缓存和失败重传；不一定需要本地 NPU |
| Ask this PDF / Ask my notes | Wi-Fi、eMMC、基础 CPU/RAM | 本地向量索引、轻量 NPU | 云端问答可先不依赖 NPU；隐私/离线模式才推高端侧算力 |
| 手写转文字 | 笔输入、触控/电磁笔、CPU、RAM | SoC NPU、本地 OCR/手写模型 | 实时笔迹必须本地；识别可本地或云端 |
| OCR | CPU、RAM、本地存储 | SoC NPU、摄像头 | 如果只处理 PDF，可云端 OCR；如果要拍照扫描，需要额外摄像头；首轮暂不作为默认配置 |
| 离线 AI/隐私模式 | 本地存储、较大 RAM | SoC NPU 或外挂 AI 加速器、散热片 | 这是增加 NPU 的主要理由之一 |
| 蓝牙笔/键盘/耳机 | Bluetooth、天线 | BLE 低功耗优化 | 蓝牙不只是配件，也可用于手机 App 配网 |

## 4. 核心硬件规格清单

本节按规格找供应商，不按厂商找方案。后续淘宝/1688、模组厂、方案商、核心板厂都应按这些参数逐项问价、要规格书、要样品和测试板。

### 4.1 屏幕规格

先按可买/可打样的屏幕规格档位理解，不把供应商或竞品名称写进选型规格。

| 优先级 | 可选屏幕规格 | 典型分辨率/显示能力 | 适合我们什么版本 | 主要取舍 |
|---|---|---|---|---|
| P0 | 10.3 英寸彩色滤光层电子纸 | 黑白约 2480 x 1860、300ppi；彩色约 1240 x 930、150ppi；约 4096 色 | 首轮原型和主力规格，适合 PDF 阅读、彩色批注、笔记 | 供应和方案成熟度相对高；彩色偏淡、底色偏暗，需要前光和盖板调校 |
| P1 | 11-12 英寸全彩电泳电子纸 | 约 11.8 英寸，2160 x 1620，约 229ppi，约 20,000 色级别 | 高端纸质感版本，适合做旗舰标注本 | 纸质感和颜色上限更高；供应、驱动、刷新和成本要重点验证 |
| P2 | 13.3 英寸彩色滤光层电子纸 | 黑白约 3200 x 2400、300ppi；彩色约 1600 x 1200、150ppi；约 4096 色 | 大屏文档版，适合论文、合同、A4 接近尺寸阅读 | 阅读面积大；重量、电池、成本、握持负担更高 |
| P3 | 7.3-7.8 英寸彩色电子纸 | 7.3 英寸约 1696 x 954、264ppi、约 20,000 色级别；7.8 英寸档常见彩色约 150ppi | 便携随身版或后续小屏产品，不作为当前主力 | 便携；但 PDF 标注空间不够，高端知识本体验受限 |
| P4 | 10.3 英寸黑白电子纸 | 约 1872 x 1404、226ppi；16 灰阶 | 彩色供应或成本不成立时的降级备选 | 纸质感、续航、成本更稳；但不满足彩色标注方向 |
| 排除 | 低频刷新全彩展示电子纸 | 面向零售标牌、广告、桌面展示 | 不适合作为手写标注本主屏 | 色彩展示好，但刷新和交互不适合高频书写 |

首轮更清晰的屏幕选择顺序：

1. 先找 `10.3 英寸彩色滤光层电子纸 + 触控/笔 + 前光 + 驱动板`，这是最现实的原型组合。
2. 同时问 `11-12 英寸全彩电泳电子纸` 是否有可采购模组和完整驱动方案，用来判断能否做高端版本。
3. 再看 `13.3 英寸彩色滤光层电子纸`，作为大屏文档版备选，不放在第一原型优先级。

向供应商询价时，屏幕部分至少要拿到这些字段：

| 字段 | 为什么必须要 |
|---|---|
| 尺寸、AA 区、外形尺寸、厚度、重量 | 决定整机尺寸、电池空间和重量 |
| 黑白分辨率/PPI、彩色分辨率/PPI | 决定文字阅读和颜色批注是否清楚 |
| 色彩数量、灰阶数量 | 决定彩色标注、图表、扫描 PDF 的表现 |
| 是否带前光、前光功耗、色温/冷暖光 | 彩色电子纸底色偏暗，前光通常影响可读性 |
| 是否带触控层、支持哪类笔 | 决定能否做标注本，而不是单纯阅读屏 |
| 刷新模式：全刷、局刷、快速黑白、彩色刷新 | 决定翻页、写字、橡皮擦和残影控制 |
| 屏幕接口、驱动板接口、参考原理图 | 决定主控能不能接，原型能不能快速点亮 |
| 波形文件和授权 | 没有匹配波形，刷新、残影、温度表现都会失控 |
| 样品价格、MOQ、交期、是否可量产 | 决定这个规格是不是只能 demo，还是能进入产品方案 |

### 4.2 触控与笔规格

| 优先级 | 可选触控/笔规格 | 典型规格 | 适合我们什么版本 | 主要取舍 |
|---|---|---|---|---|
| P0 | 无源电磁笔 + 触控层 | 无需给笔充电；4096 级压感起；报点率 >= 120Hz；支持掌托误触 | 首轮标注本原型和主力版本 | 纸笔感和续航更好；需要屏幕、触控层、主控三方适配 |
| P1 | 主动电容笔 + 电容触控层 | 笔内置电池或充电；4096/8192 级压感；可支持侧键/倾斜 | 如果供应更容易、模组更完整，作为备选 | 模组集成容易；笔需要充电，纸笔感和延迟要实测 |
| P2 | 电容触控 + 手指操作，不配主动笔 | 多点触控；无压感；只能做简单高亮和手势 | 阅读器/低配版本 | 成本低；不适合高端手写标注 |
| P3 | 高阶手写笔规格 | 8192 级压感；>= 200Hz 报点；倾斜；尾擦；低初始激活力 | 高端旗舰版本 | 体验上限高；成本、调校、供应链复杂度更高 |

首轮更清晰的触控与笔选择顺序：

1. 先找 `无源电磁笔 + 10.3 英寸彩色电子纸触控层 + 掌托误触`，验证纸上写字的基础体验。
2. 同时问 `主动电容笔方案`，作为供应链和成本备选。
3. 暂不把纯手指触控作为主力，因为它不符合高端标注本定位。

询价时，触控与笔至少要拿到这些字段：

| 字段 | 为什么必须要 |
|---|---|
| 笔类型：无源电磁/主动电容/纯电容触控 | 决定笔是否要充电、屏幕模组复杂度和书写体验 |
| 压感级数、初始激活力 | 决定轻写是否出墨、线条是否自然 |
| 报点率、坐标准确度、边缘精度 | 决定快速写字、签名、边缘批注是否稳定 |
| 笔迹延迟、触控延迟 | 决定是否像纸笔，不能只看屏幕刷新 |
| 视差、盖板厚度、贴合方式 | 决定笔尖和笔迹是否对得上 |
| 掌托误触策略 | 标注本必须支持手掌放在屏幕上写字 |
| 侧键、尾擦、倾斜 | 决定高端笔记工作流是否完整 |
| 笔尖材质、寿命、替换成本 | 决定摩擦感和长期耗材成本 |

### 4.3 主控规格

| 优先级 | 可选主控规格 | 典型规格 | 适合我们什么版本 | 主要取舍 |
|---|---|---|---|---|
| P0 | 四核低功耗应用处理器 | 4x A53/A55 级别；2-4GB RAM；32-64GB 存储；USB-C；Wi-Fi/BT 可集成 | 首轮原型和主力标注本 | 性能足够跑阅读、笔记、同步；功耗可控 |
| P1 | 四核/六核低功耗应用处理器 + 更大内存 | A55 级别或同档；4GB RAM；64GB 存储起 | 更流畅的 PDF 和云端 AI 工作流 | 成本略高，但更稳妥 |
| P2 | 八核大小核高性能处理器 | 大核 + 小核；6-8GB RAM；64-128GB 存储 | 大屏、多任务、本地 AI 原型 | 性能强；功耗、温升、成本可能过高 |
| P3 | 工业长期供货处理器 | 4 核 A53/A55 级；长生命周期；BSP 稳定 | 企业版或长期供货版本 | 供货稳；成本和生态可能不如消费级平台 |

首轮更清晰的主控选择顺序：

1. 先找 `四核低功耗应用处理器 + 4GB RAM + 64GB 存储 + USB-C + Wi-Fi/BT` 的开发板或核心板。
2. 要求能接目标电子纸驱动板，并能跑 PDF、笔迹、同步、休眠唤醒。
3. 高性能八核平台只作为压力测试和本地 AI 备选，不作为第一版默认。

询价时，主控至少要拿到这些字段：

| 字段 | 为什么必须要 |
|---|---|
| CPU 核心、主频、制程、典型功耗 | 决定 PDF、笔迹和系统流畅度，也决定续航 |
| RAM 容量和类型 | 决定大 PDF、多文档、云同步缓存是否卡顿 |
| 存储容量和类型 | 决定本地文件、离线缓存、笔记数量 |
| 电子纸接口支持 | 决定是否能接驱动板、局刷和全刷是否可控 |
| USB-C 能力 | 决定充电、文件传输、恢复模式、调试便利性 |
| 低功耗模式 | 决定待机续航，必须看 suspend/resume 和唤醒时间 |
| BSP、内核、系统支持 | 决定开发成本和驱动稳定性 |
| I2C/SPI/UART/GPIO/PWM/ADC 数量 | 决定触控笔、前光、电量计、按键等外设能否接全 |

### 4.4 通信规格

| 优先级 | 可选通信规格 | 典型规格 | 适合我们什么版本 | 主要取舍 |
|---|---|---|---|---|
| P0 | Wi-Fi 5 + BLE 5.x | 2.4GHz/5GHz；BLE 配网；支持蓝牙键盘/配件 | 首轮原型和主力版本 | 能覆盖同步、OTA、云端 AI；成本和功耗可控 |
| P1 | Wi-Fi 6 + BLE 5.x | 更好的弱网和拥挤网络表现 | 高端版本 | 成本略高，软件适配和认证资料要确认 |
| P2 | Wi-Fi + BLE + 可插拔蜂窝模块预留 | 不默认装蜂窝，只预留接口和空间 | 企业版/海外弱网场景备选 | 功耗、认证、资费复杂，不进首轮默认 BOM |
| 排除 | 只做离线 USB 传输 | 无 Wi-Fi 或只有 USB | 不适合本项目 | 无法支撑云同步、OTA、云端 AI |

首轮更清晰的通信选择顺序：

1. 先找 `双频 Wi-Fi + BLE 5.x`，通信必须能支持云同步和 OTA。
2. 蓝牙优先用于配网、键盘/外设连接，不把蜂窝放进第一版。
3. 如果目标是企业版或弱网地区，再评估蜂窝/eSIM 的空间、认证和功耗。

询价时，通信至少要拿到这些字段：

| 字段 | 为什么必须要 |
|---|---|
| Wi-Fi 协议和频段 | 决定海外家庭/办公室网络兼容性 |
| BLE 版本和 Profile | 决定配网、键盘、配件连接能力 |
| 模组接口 | 决定主控接入方式和 PCB 难度 |
| 天线类型和净空要求 | 决定实际信号和整机布局 |
| 待机功耗和唤醒重连时间 | 决定云同步是否拖垮续航 |
| 认证资料 | 决定后续出海认证难度 |

### 4.5 AI/算力规格

这里不要只看 TOPS。TOPS 只能说明 INT8 峰值算力，真实表现还取决于模型是否能编译、算子是否支持、内存带宽、预处理是否走 CPU、驱动成熟度和散热限制。

| 优先级 | 可选算力规格/芯片举例 | 可跑模型举例 | 可支持的任务/动作 | 预期速度和表现 | 理论功耗 |
|---|---|---|---|---|---|
| P0 | 无独立 AI 加速器，CPU + 云端 AI；芯片例：四核 A55/A53 级主控、RK3566/RK3568 这类低功耗应用处理器 | 本地只跑规则算法、图像压缩、PDF 预处理、很小的分类模型；大模型、摘要、问答全部云端 | 上传 PDF/笔记、云端总结、云端问答、云端 OCR；本地做文件切页、缩略图、压缩、缓存 | 云端 AI 体验主要取决于网络和服务端，常见交互是数秒级；本地 CPU 跑整页 OCR 会慢，不采用作为主体验 | 额外 AI 功耗最低；CPU 短时负载约 0.5-2.5W，待机不增加独立芯片功耗 |
| P1 | 轻量内置 NPU，约 1-3 TOPS；芯片例：NXP i.MX 8M Plus（2.3 TOPS）、RK3566/RK3568（约 0.8 TOPS，低于本档但可作轻量对照） | 小型 OCR 检测/识别模型、手写行识别模型、小型图像分类模型、文档方向/版面分类模型 | 本地识别单行手写、局部 OCR、图片清理、文档旋转检测、云端上传前预处理 | 单行/局部识别可做到几十到数百毫秒；整页 OCR 可能 1-5 秒，复杂 PDF 仍要求云端 | NPU 运行时通常约 0.5-2W 额外功耗；连续运行会明显影响续航 |
| P2 | 中等内置 NPU，约 5-6 TOPS；芯片例：RK3576、RK3588、Kendryte K230 | 整页 OCR 检测 + 识别、小型手写识别模型、轻量版面分析模型、小型 embedding/rerank 模型 | 本地整页 OCR、离线手写转文字、笔记分类、文档切块预处理、弱网下基础识别 | 整页 OCR 有机会做到约 0.5-2 秒/页；手写行识别更接近实时；能明显降低云端调用次数 | SoC + NPU 高负载约 1.5-4W；需要测温升和限频 |
| P3 | 外挂 AI 加速器，约 10-13 TOPS；模块例：Hailo-8L、Raspberry Pi AI HAT+ 13 TOPS 档 | 多个 OCR/视觉模型流水线、小型视觉分类/检测模型、轻量文档理解模型 | 离线 OCR、离线版面分析、批量文档预处理、企业隐私模式 | 多模型流水线比内置 NPU 更稳；文档视觉任务可接近实时，但对电子纸交互通常过剩 | 加速器模块约 2-5W，再加主控功耗；薄设备需要评估散热和电池 |
| P4 | 高算力外挂 AI，加速器约 20-30 TOPS；模块例：Hailo-8、Raspberry Pi AI HAT+ 26 TOPS 档 | 多模型并发视觉流水线、更大的文档视觉模型、小型 VLM/多模态前处理；不采用指望本地完整大语言模型 | 企业离线处理、批量扫描/OCR、多文档本地预处理、隐私版高配 | 可跑更复杂的视觉流水线，但电子纸刷新慢，很多性能无法转化成交互体验 | 加速器典型约 2.5-6W，整机峰值更高；不适合首轮轻薄消费版 |

首轮更清晰的算力选择顺序：

1. 第一原型先按 `无独立 AI 加速器` 做，确保云端 AI、同步、笔记主链路跑通。
2. 同时选择一个 `2-6 TOPS 内置 NPU` 主控作为对照样板，验证本地 OCR、手写识别、文档预处理是否值得上。
3. 外挂 AI 只作为企业离线版本研究，不进消费级第一版 BOM。
4. 不把本地完整 LLM/VLM 当作第一版目标。电子纸设备的本地 AI 更适合 OCR、手写识别、分类、版面分析、向量/检索预处理。

询价时，算力至少要拿到这些字段：

| 字段 | 为什么必须要 |
|---|---|
| 是否有 NPU、INT8 TOPS | 判断本地 OCR/手写识别有没有硬件基础 |
| 支持模型格式 | 决定模型能不能部署，不只看 TOPS |
| SDK、编译工具链、示例模型 | 决定开发难度 |
| 运行内存占用 | 决定 RAM 配置是否要提高 |
| 推理功耗、持续运行温升 | 决定是否会破坏电子纸长续航定位 |
| 与主控接口 | 外挂 AI 时决定 PCB 和驱动复杂度 |

### 4.6 电源规格

| 优先级 | 可选电源规格 | 典型规格 | 适合我们什么版本 | 主要取舍 |
|---|---|---|---|---|
| P0 | 10.3 英寸主力电池规格 | 3700-4500mAh，约 14-17Wh；USB-C 5V/2A 起 | 首轮原型和主力版本 | 厚度、重量、续航比较平衡 |
| P1 | 11-12 英寸高端电池规格 | 4800-5500mAh，约 18-21Wh | 高端纸质感版本 | 续航更稳；重量和厚度增加 |
| P2 | 13.3 英寸大屏电池规格 | 5500-7000mAh，约 21-27Wh | 大屏文档版 | 能支撑大屏；重量、充电时间上升 |
| P3 | 轻薄优先电池规格 | 3000-3700mAh，约 11-14Wh | 轻薄测试机或低配版 | 更薄更轻；续航风险更高 |

首轮更清晰的电源选择顺序：

1. 先找 `4000-5000mAh 超薄软包电池 + 保护板 + fuel gauge + USB-C 充电`。
2. 用 Wh 统一比较，不只看 mAh。
3. 同时确认前光、Wi-Fi、书写、本地 AI 峰值电流是否会导致掉电或重启。

询价时，电源至少要拿到这些字段：

| 字段 | 为什么必须要 |
|---|---|
| mAh、标称电压、Wh | Wh 才能比较真实能量 |
| 电池尺寸、厚度、重量 | 决定整机厚度和手感 |
| 保护板规格 | 决定过充、过放、短路安全 |
| 充电 IC、充电功率 | 决定充电时间和温升 |
| fuel gauge | 决定电量显示是否可信 |
| PMIC 电源轨 | 决定屏幕、主控、前光、无线是否能分域供电 |
| 休眠电流、峰值电流 | 决定长待机和高负载稳定性 |

电池统一按 Wh 估算：

```text
电池 Wh = 电池容量 Ah x 标称电压 V

日耗电 = 待机 + 屏幕刷新 + 前光 + Wi-Fi/蓝牙 + 书写 + 本地 AI + 传感器

目标电池 Wh = 日耗电 x 目标续航天数 / 可用电量比例
```

### 4.7 安全与存储规格

| 优先级 | 可选安全/存储规格 | 典型规格 | 适合我们什么版本 | 主要取舍 |
|---|---|---|---|---|
| P0 | 64GB 本地存储 + 系统级加密 | 64GB eMMC/UFS；文件加密；OTA 签名校验 | 首轮原型和主力版本 | 成本可控，能覆盖 PDF、笔记和缓存 |
| P1 | 128GB 本地存储 + 完整安全启动 | 128GB 存储；安全启动；OTA 回滚；数据擦除 | 高端版本 | 更适合大量 PDF；成本增加 |
| P2 | 64/128GB + TEE/安全存储 | token、密钥、账号凭据放入安全区域 | 云端 AI 和账号体系版本 | 隐私和账号安全更好；开发复杂度增加 |
| P3 | 独立安全芯片 | 独立密钥存储、设备身份、证书保护 | 企业版或高合规版本 | 安全上限高；BOM 和软件集成更复杂 |

首轮更清晰的安全与存储选择顺序：

1. 先按 `64GB 存储 + 系统级文件加密 + OTA 签名校验 + 数据擦除` 做主力规格。
2. 如果目标用户有大量 PDF/资料库，再评估 128GB。
3. 独立安全芯片先不进第一版，除非明确要企业级设备身份或高合规要求。

询价时，安全与存储至少要拿到这些字段：

| 字段 | 为什么必须要 |
|---|---|
| 存储容量、类型、读写速度、寿命 | 决定文件容量、系统流畅度和可靠性 |
| 文件加密能力 | 决定用户笔记和 PDF 的本地隐私 |
| 安全启动 | 防止固件被随意替换 |
| OTA 签名和失败回滚 | 防止升级失败变砖 |
| 数据擦除方式 | 恢复出厂后不能残留用户文件 |
| token/密钥存储方式 | 决定云端账号凭据是否安全 |
| 调试口关闭策略 | 量产设备不能通过调试口直接读数据 |
