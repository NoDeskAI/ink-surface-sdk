## 关键结论

1. 项目文档以本地 Markdown 为主编辑源，飞书作为协作查看和同步目标。
2. H2 里程碑按“6 月底方案/采购/软件原型、7 月中旬 MVP、7 月底硬件选型、8 月底样机验证、9 月底最终实测、11 月市场推广、12 月复盘”执行。
3. 硬件采购数据落到飞书多维表格，本地保留字段设计、视图规则、样例行和判断口径。
4. 技术方案目录需要保留主线总览，并把零散技术文档按产品、软件、硬件、Schema/Adapter、样机验证归类。
5. 同步到飞书时使用官方 Markdown 导入，确保表格渲染为飞书原生表格，正文不重复显示页面标题。
