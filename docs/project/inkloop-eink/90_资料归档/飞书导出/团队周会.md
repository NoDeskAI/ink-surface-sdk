# 团队周会

## 归档说明

本文来自早期飞书导出资料，已改写为正式归档记录。当前项目周会以 `03_会议纪要/周会纪要模板.md` 和实际周会纪要为准，本文件仅保留历史来源和会议机制参考。

## 周会机制

| 项目 | 内容 |
| --- | --- |
| 会议目的 | 同步出海墨水屏项目的进度、风险、决策和下周动作 |
| 固定频率 | 每周一次 |
| 固定输入 | 项目总看板、风险与问题日志、硬件采购清单、技术方案总览 |
| 固定输出 | 本周完成、风险阻塞、下周计划、需要决策的问题 |

## 标准议程

| 顺序 | 议题 | 输出 |
| --- | --- | --- |
| 1 | 上周任务完成情况 | 完成项、延期项、原因 |
| 2 | 软件原型和 MVP 进展 | 演示结果、问题清单 |
| 3 | 硬件采购和样机进展 | 到货、测试、风险 |
| 4 | 市场与运营准备 | 素材、线索、用户反馈 |
| 5 | 本周决策 | 决策记录和负责人 |
| 6 | 下周行动 | 截止时间和验收标准 |

## H2 周会关注点

| 阶段 | 周会重点 |
| --- | --- |
| 6 月 | 方案整理、软件原型、采购清单 |
| 7 月 | MVP 演示、硬件选型、采购到货 |
| 8 月 | 样机装配、功能验证、问题关闭 |
| 9 月 | 最终样机和最终软件实测 |
| 10-11 月 | 海外推广素材、线索、访谈和运营数据 |
| 12 月 | H2 复盘和下一阶段决策 |
