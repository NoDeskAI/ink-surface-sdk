# 每日进展同步

## 归档说明

本文来自早期飞书导出资料，已改写为正式归档记录。当前项目不强制每日开会，日常进展以本地文档、项目总看板和周会纪要沉淀为主。

## 日常同步口径

| 场景 | 同步内容 | 记录位置 |
| --- | --- | --- |
| 关键任务完成 | 完成内容、验收结果、关联文档 | 项目总看板或周会纪要 |
| 出现阻塞 | 问题、影响、处理动作、负责人 | 风险与问题日志 |
| 硬件采购变化 | 物料、供应商、价格、交期、状态 | 硬件采购清单 |
| 软件原型变化 | 版本、演示路径、已知问题 | 技术方案或原型说明 |
| 市场反馈 | 线索、访谈、转化、素材表现 | 项目总看板和运营记录 |

## 每日更新格式

| 字段 | 填写口径 |
| --- | --- |
| 日期 | YYYY-MM-DD |
| 今日完成 | 写可验证结果，不写过程流水 |
| 当前风险 | 只记录会影响里程碑的问题 |
| 明日重点 | 不超过 3 项 |
| 需要决策 | 明确决策选项、影响和截止时间 |

## 当前适用节奏

6-7 月属于方案、采购和 MVP 集中推进阶段。日常同步只记录会影响 6 月底软件原型、7 月中旬 MVP、7 月底硬件选型的事项；普通过程信息进入周会纪要即可。
