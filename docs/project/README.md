# Project Documents

This directory stores product/project documents that are tied to InkLoop product strategy, launch scope, and roadmap decisions beyond the reusable SDK/package layer.

## Directories

| Directory | Purpose |
| --- | --- |
| `inkloop-ai-pen-kickstarter/` | Current 2026-10 Kickstarter project baseline: AI Pen, Capture Surface, education + business meeting launch scope, InkGraph contracts, GTM, risks, and gates |
| `inkloop-eink/` | Historical/local-first e-paper project knowledge base and second product loop for InkLoop Paper |

## Maintenance

- Keep stable system architecture and implementation contracts in `docs/architecture.md`, `docs/ink-surface-sdk.md`, and related root docs.
- Keep current Kickstarter product goals, milestones, project management, hardware procurement, meeting/business/education scope, and full project planning under `docs/project/inkloop-ai-pen-kickstarter/`.
- Keep e-paper-specific or historical materials under `docs/project/inkloop-eink/`, and promote only reusable runtime decisions into root docs.
- When a project document affects reusable package contracts or implementation, promote the stable technical decision back into the root docs or an execution plan.
