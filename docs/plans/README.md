# Plans

This directory stores CE execution plans.

Naming convention:

```text
YYYY-MM-DD-NN-feat-xxx-plan.md
```

Plans should usually include:

- overview
- problem frame
- requirements trace
- scope boundaries
- context and research
- technical decisions
- implementation units
- verification strategy
- risks and dependencies

Plans describe how to do upcoming or in-progress work. After implementation, recurring lessons should be extracted into `docs/solutions/`.
